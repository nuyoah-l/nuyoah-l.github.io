问题背景

在使用 Hexo 搭建博客（nexmoe 主题）时，遇到了封面图尺寸不统一的问题：

  • 电脑端封面显示正常
  • 移动端封面高度过小，图片被压缩在一个很窄的区域里

排查过程

第一步：检查生成的 HTML 结构

第二步：计算移动端实际高度

以手机屏幕宽度 375px 为例：

  • padding-top: 28.125% → 实际高度 = 375 × 28.125% ≈ 105px
  • 这个高度确实太小，导致封面区域被严重压缩

第三步：定位问题根源

  • ✅ EJS 模板生成的内联样式是正确的
  • ✅ 电脑端显示正常
  • ❌ 移动端 padding-top 比例太小，需要单独调整

解决方案

核心思路

  • 电脑端：保持原有比例（28.125%）
  • 移动端：通过 CSS 媒体查询覆盖为更大的比例

2. 图片懒加载不影响尺寸

loading="lazy" 本身不会导致尺寸问题，但必须配合 width/height 属性或 padding-top 预留空间，否则会引起布局偏移（CLS）。

3. 电脑端正常但移动端异常的排查方法

  • 使用浏览器开发者工具（F12）切换到移动设备视图
  • 检查 Computed 样式中的 padding-top、min-height、height
  • 查看是哪个 CSS 规则或内联样式生效

总结

问题 原因 解决方案
移动端封面太小 padding-top 28.125% 产生高度仅 105px 媒体查询覆盖为 50%-56.25%
封面比例不统一 图片宽高比不一致 统一使用 1280:720 (16:9)
min-height 冲突 主题 JS 动态添加 CSS 强制覆盖

最终效果

  • ✅ 电脑端封面比例适中
  • ✅ 移动端封面高度明显增大，视觉效果更好
  • ✅ 不同设备自适应，无需修改 HTML 结构
  • ✅ 代码简洁，维护方便
